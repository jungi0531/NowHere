# Git / PR / Issue 컨벤션

## 1. Issue 규칙

### 제목 형식

[type] 작업 내용

type:

* feat: 기능
* fix: 버그
* chore: 설정/환경

예:
[feat] check-in 상태 흐름 연결

### 본문 템플릿

## 목적

이 작업을 하는 이유

## 작업 내용

* [ ] 작업 1
* [ ] 작업 2

## 완료 조건

* [ ] 완료 기준

---

## 2. 브랜치 규칙

* feature/기능명
* fix/버그명
* chore/환경설정

---

## 3. 커밋 규칙

형식: <type>: <설명> (#이슈번호)

예:
feat: check-in 상태 연결 (#2)

추가 규칙:

* 한 PR을 하나의 커밋으로 퉁치지 말고 기능 개발 단위로 커밋한다.

---

## 4. PR 규칙

* 반드시 이슈 기반으로 생성
* PR 제목 = 커밋 규칙과 동일
* PR 본문에 목적/변경내용/영향범위 작성

---

## 5. 개발 원칙

* PR 단위로 작게 나눈다
* verify 통과가 기준이다
* 과도한 추상화 금지
